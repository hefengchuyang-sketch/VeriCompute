```
│  Layer 0: Online Rewards (Passive)         │  0.5-2 coins/hour
│  优先级：保底 | 价值：稳定收入              │
└─────────────────────────────────────────────┘
```

## 🔬 Layer 1: S-Box挖矿

### 什么是S-Box？

**S-Box（Substitution Box）** 是密码学中的核心组件：
- AES加密算法的核心
- 提供非线性变换
- 抵抗差分和线性攻击

### 为什么挖S-Box？

1. **真实价值**：高质量S-Box可用于实际密码系统
2. **学术贡献**：推动密码学研究
3. **稀缺性**：好的S-Box很难找到
4. **可验证**：质量可以客观评估

### S-Box质量评估

```python
# 三个核心指标
1. 非线性度（Nonlinearity）：≥100
2. 差分均匀性（Differential Uniformity）：≤8
3. 雪崩效应（Avalanche Effect）：≥0.45

# 综合评分
score = 0.4×nonlinearity + 0.3×(1/differential) + 0.3×avalanche
```

### S-Box挖矿流程

```
1. 矿工接收S-Box任务
   ↓
2. 生成候选S-Box（随机或启发式）
   ↓
3. 评估质量（非线性度、差分均匀性、雪崩效应）
   ↓
4. 如果达标 → 提交到链上
   ↓
5. 网络验证 → 获得奖励（52-98币）
```

### 实际应用

挖出的高质量S-Box可以：
- 用于新的加密算法设计
- 提交到密码学期刊
- 授权给商业密码产品
- 建立S-Box数据库

## 💼 Layer 2: POUW挖矿

### 什么是POUW？

**POUW（Proof of Useful Work）** = 有用工作量证明

不是计算无意义的哈希，而是执行真实的计算任务。

### 支持的任务类型

1. **AI推理**
   - GPT模型推理
   - Stable Diffusion图像生成
   - 语音识别/合成

2. **视频渲染**
   - Blender 3D渲染
   - FFmpeg视频转码
   - 特效处理

3. **科学计算**
   - 矩阵运算
   - 物理模拟
   - 气候建模

4. **数据处理**
   - ETL数据转换
   - 大数据分析
   - 机器学习训练

### POUW挖矿流程

```
1. 用户提交任务 + 预算
   ↓
2. 任务进入POUW池
   ↓
3. 矿工选择匹配任务
   ↓
4. 执行计算 → 提交结果
   ↓
5. 网络验证 → 获得奖励（30-80币）
   ↓
6. 用户获得计算结果
```

### 任务评分机制

```python
score = 0.30×complexity      # 计算复杂度
      + 0.25×verifiability   # 可验证性
      + 0.25×value           # 实际价值
      + 0.15×utilization     # 资源利用率
      - 0.05×cheat_risk      # 作弊风险
```

## ⚙️ Layer 3: PoW保底挖矿

### 为什么需要PoW？

**保证链活性**：当没有S-Box或POUW任务时，链仍然能继续运行。

### PoW特点

- **传统哈希挖矿**：找到满足难度的nonce
- **奖励最低**：10-20币/块
- **难度动态调整**：保持30秒出块时间
- **保底机制**：确保链不停

### PoW挖矿流程

```
1. 检查S-Box池 → 空
   ↓
2. 检查POUW池 → 空
   ↓
3. 启动PoW挖矿
   ↓
4. 计算哈希直到找到有效nonce
   ↓
5. 提交区块 → 获得奖励（10-20币）
```

## 🎁 Layer 0: 在线奖励池

### 为什么需要在线奖励？

**解决小矿工困境**：算力低 → 挖不到块 → 0收益 → 退出

### 在线奖励特点

- **被动收入**：只要在线就有收益
- **稳定可预测**：0.5-2币/小时
- **板块币发放**：不是MAIN
- **资金可追溯**：所有来源透明

### 收益计算

```python
reward = pool_balance × (contribution_score / total_contribution)

contribution_score = compute_power × online_hours × stake_weight
```

## 🔄 共识选择策略

### 矿工视角

```python
def select_mining_task(miner):
    # 优先级1：检查S-Box池
    if has_sbox_task():
        return mine_sbox()  # 最高奖励
    
    # 优先级2：检查POUW池
    if has_pouw_task():
        return mine_pouw()  # 高奖励
    
    # 优先级3：PoW保底
    return mine_pow()  # 基础奖励
    
    # 同时：在线奖励持续累积
```

### 网络视角

**理想分布**：
- S-Box: 50%
- POUW: 35%
- PoW: 15%

**动态调整**：
- 如果S-Box过多 → 提高难度
- 如果POUW过少 → 增加任务补贴
- 如果PoW过多 → 说明任务不足，需要吸引更多用户

## 💰 收益对比

### 场景1：大矿工（8张H100）

```
每天挖矿时间：24小时
算力：800 TFLOPs

S-Box挖矿（假设50%时间）：
  12小时 × 2块/小时 × 75币/块 = 1,800币

POUW挖矿（假设35%时间）：
  8.4小时 × 2块/小时 × 55币/块 = 924币

PoW挖矿（假设15%时间）：
  3.6小时 × 2块/小时 × 15币/块 = 108币

在线奖励（24小时）：
  24小时 × 1.5币/小时 = 36币

总收益：1,800 + 924 + 108 + 36 = 2,868币/天
```

### 场景2：小矿工（1张RTX4090）

```
每天挖矿时间：24小时
算力：100 TFLOPs

挖矿收益（运气不好，没挖到块）：0币

在线奖励（24小时）：
  24小时 × 1.0币/小时 = 24币

总收益：24币/天（保底）
```

### 场景3：中等矿工（2张RTX4090）

```
每天挖矿时间：24小时
算力：200 TFLOPs

POUW挖矿（运气好，挖到2个块）：
  2块 × 55币/块 = 110币

在线奖励（24小时）：
  24小时 × 1.2币/小时 = 28.8币

总收益：110 + 28.8 = 138.8币/天
```
